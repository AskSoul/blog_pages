Git 工作流完全指南 ​

概述 ​

Git 是现代软件开发中不可或缺的版本控制工具。本文将介绍 Git 高级操作，详解 Git Flow、GitHub Flow 等工作流，帮助团队提升协作效率。

Git 基础回顾 ​

核心概念 ​

工作区 (Working Directory)
    ↓ git add
暂存区 (Staging Area)
    ↓ git commit
本地仓库 (Local Repository)
    ↓ git push
远程仓库 (Remote Repository)

分支类型 ​

分支用途命名规则
master生产环境master/main
develop开发主分支develop
feature功能开发feature/*
release发布准备release/*
hotfix紧急修复hotfix/*

完整流程 ​

bash
# 1. 从 develop 创建功能分支
git checkout -b feature/user-auth develop

# 2. 开发完成后，合并到 develop
git checkout develop
git merge --no-ff feature/user-auth

# 3. 删除功能分支
git branch -d feature/user-auth

# 4. 创建发布分支
git checkout -b release/1.0 develop

# 5. 发布后合并到 master 和 develop
git checkout master
git merge release/1.0
git tag -a v1.0 -m "版本 1.0"

git checkout develop
git merge release/1.0

# 6. 删除发布分支
git branch -d release/1.0

GitHub Flow ​

适用场景 ​

  • 持续部署的项目
  • 团队规模较小
  • 无多版本并行开发需求

流程 ​

bash
# 1. 创建特性分支
git checkout -b feature/dark-mode

# 2. 开发并提交
git commit -m "添加深色模式支持"

# 3. 推送分支
git push -u origin feature/dark-mode

# 4. 创建 Pull Request

# 5. 代码审查后合并
# 通过 GitHub 界面合并

# 6. 删除分支
git branch -d feature/dark-mode
git push origin --delete feature/dark-mode

高级操作 ​

变基 (Rebase) ​

bash
# 变基到主分支
git checkout feature/login
git rebase main

# 解决冲突后继续
git rebase --continue

# 中止变基
git rebase --abort

# 交互式变基（修改提交历史）
git rebase -i HEAD~3

重置 (Reset) 与还原 (Revert) ​

bash
# 重置到指定提交（修改历史）
git reset --soft HEAD~1    # 保留更改在暂存区
git reset --mixed HEAD~1   # 保留更改在工作区
git reset --hard HEAD~1    # 丢弃更改

# 还原指定提交（安全操作）
git revert <commit-hash>

Pull Request 规范 ​

  1. 保持 PR 小而专注
  2. 编写清晰的描述
  3. 关联相关 issue
  4. 要求代码审查
  5. 通过所有 CI 检查
